Glosario

N | O | P | Q | R | S | T | U | V | W | X | Y | Z

N

N – Node.js Entorno de ejecución para JavaScript del lado del servidor, que permite construir aplicaciones escalables y rápidas.

O

O – Open Source (Código Abierto) Modelo de desarrollo en el cual el código fuente del software está disponible para ser estudiado, modificado y distribuido libremente.

P

P – Protocolo Conjunto de reglas que permiten la comunicación entre dispositivos o aplicaciones, como HTTP o FTP.

Q

Q – Query Consulta realizada a una base de datos para obtener o manipular información, comúnmente usando SQL.

R

R – Repositorio Ubicación central donde se almacena y gestiona el código fuente de un proyecto, a menudo en plataformas como GitHub o GitLab.

S

S – Scrum Marco de trabajo ágil utilizado para el desarrollo y gestión de proyectos de software, basado en iteraciones llamadas “sprints”.

T

T – Test Unitario Prueba automatizada que verifica que una unidad mínima de código (como una función) se comporta como se espera.

U

U – UML (Unified Modeling Language) Lenguaje de modelado usado para visualizar, especificar y documentar sistemas de software.

V

V – Variable Espacio de almacenamiento en la memoria de un programa que contiene un valor que puede cambiar durante la ejecución.

W

W – Web Service Servicio disponible en internet que permite la comunicación entre aplicaciones mediante protocolos como SOAP o REST.

X

X – XML (eXtensible Markup Language) Lenguaje de marcado que permite estructurar datos de manera legible tanto para humanos como para máquinas.

Y

Y – YAML (YAML Ain’t Markup Language) Formato de serialización de datos legible por humanos, usado comúnmente en archivos de configuración.

Z

Z – Zip Formato de compresión utilizado para reducir el tamaño de archivos o agrupar varios archivos en uno solo.
